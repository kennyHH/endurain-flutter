## Summary

This PR ships Endurain Mobile `v0.0.33` and consolidates the practical improvements delivered after `v0.0.32` with focus on upload reliability and start-readiness UX.

## What changed since v0.0.32

- Standardized GPX export/upload filename format to `YYYY-MM-DD_HH-mm_Type[_Name].gpx`.
- Improved start UX: `Start tracking` stays visible, is disabled until stable GPS fix, and shows explicit readiness guidance.
- Added upload visibility metadata variants (`hidden=false`, `visibility=public`, etc.) for better server-default compatibility.
- Added request idempotency headers (`Idempotency-Key`, `X-Upload-Activity-Id`) for upload traceability.
- Added in-flight deduplication per `activity.id` to prevent parallel duplicate uploads.
- Persisted upload success state and added pre-upload short-circuit for already-uploaded activities.
- Ensured queue retry success persists `uploaded=true` to avoid replayed duplicates later.

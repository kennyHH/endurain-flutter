# PR: Ship Endurain Mobile v0.0.33

## Summary

This PR prepares the v0.0.33 mobile release and consolidates the user-facing and server-integration improvements delivered after v0.0.32.

Primary focus:

- more predictable upload behavior,
- clear GPS start readiness UX,
- clean GPX export/upload naming,
- reduced duplicate activity creation on server sync.

## Delta from v0.0.32 to v0.0.33

### Upload reliability and duplicate prevention

- Added explicit upload visibility metadata to improve compatibility with server visibility defaults.
- Added stable idempotency headers per activity upload.
- Added in-flight deduplication for parallel uploads of the same activity.
- Persisted upload success state in local activity storage.
- Added pre-upload guard to skip already-uploaded activities.
- Ensured queue retry success updates local upload state to avoid replays.

### Tracking start UX

- `Start tracking` remains visible but disabled until a stable GPS fix is available.
- Added clear in-context guidance so readiness is understandable before tracking start.

### GPX export naming

- Standardized GPX naming to:
  - `YYYY-MM-DD_HH-mm_Type.gpx`
  - `YYYY-MM-DD_HH-mm_Type_Name.gpx`

## QA and verification

- `flutter analyze` passes.
- `flutter test` passes.
- Upload-service regression tests include:
  - multipart metadata coverage,
  - in-flight deduplication,
  - already-uploaded short-circuit,
  - queue success persistence.
